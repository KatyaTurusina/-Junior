Собираем данные автоматизировано

Мы уже научились формировать url ссылки и узнали набор функций API к данным фильмах. Теперь необходимо автоматизировать процесс сбора данных. Акитилана Кифарг подготовила для вас краткий экскурс в язык программирования Python и необходимые библиотеки, которые могут пригодиться при написании кода. В качестве среды разработки будем использовать trinket.

Чтобы использовать библиотеки в Python, их необходимо импортировать:

import <название библиотеки>

После чего к ним можно обращаться и использовать доступные методы через точку:

<название_библиотеки>.<название_метода>(<необходимые_параметры>)

Библиотека requests

Библиотека предназначена для работы с запросами. Вспомним: чтобы получить ответ от API, нужно сформировать запрос. В нашем случае в качестве запроса мы будем передавать только ссылку url с использованием метода get() и записывать ответ в переменную r

url = '<ссылка на ресурс>'
r = requests.get(url)

Чтобы получить содержимое ответа запишем его в переменную result в формате json:

result = r.json()

Формат json

Формат json имеет структуру ключ (key) и значение (value), записанные в фигурных скобках через :. При этом в качестве значений может быть задана последовательность в виде списка в [] через запятую или иметь вложенность, то есть в качестве значения также выступает ключ-значение, например:

{'key':['value1', 'value2'],
 'key1':'value',
 'key2':{'key3':'value'},
 'key4':[{'key5':'value'},{'key5':'value'}]
}

Чтобы получить значение по ключу, обратимся к нему в квадратных скобках:

result['key']

Если значение имеет вид списка, но может вернуться пустым, проверим это с помощью условия:

result['key']==[]

== - проверка на равенство значений,  а != - проверка на неравенство

Чтобы получить значение key5 нулевого элемента из списка записей по ключу key4:

result['key4'][0]['key5']

То есть, берем значения по ключу, вернулся список в []. Далее получаем нулевую запись из списка и только теперь обращаемся к значению.

Циклы

Циклы позволяют выполнять один и тот же участок программного кода несколько раз. Например, нам необходимо получить записи о фильмах по нескольким жанрам. Чтобы не дублировать часть кода с получением данных по API воспользуемся циклом for, где будем использовать каждый жанр в отдельности из списка жанров.  Каждый шаг цикла будет принимать значение соответствующего жанру в переменную genre.

Ввод данных пользователем осуществляется через метод input(). Ссылку сделаем шаблоном с помощью форматирования строки, для этого поставим f перед кавычками. В самой строке укажем в {} название переменной, которая будет подставляться в шаблон.

import requests

genre1 = input("Введите жанр: ")
genre2 = input("Введите другой жанр: ")

for genre in [genre1, genre2]:
  url = f'https://yupest2.pythonanywhere.com/api/v1.0/movies/?genre={genre}'
  print(url)
  r = requests.get(url)
  result = r.json()

Если мы не знаем количество шагов (последовательность значений), то будем использовать цикл while (пока). Такой цикл выполняется до тех пор, пока выполняется проверяемое условие.

while <условие>:
   pass

На данном этапе, содержимое ответа от API мы никуда не сохраняем и результат шага теряется при выполнении следующего. Исправим это с помощью сохранения записей в единый список.

Списки и сохранение результата в файл

Создадим пустую переменную films = [], в которую будем добавлять список записей. Чтобы в итоге получить единый список, воспользуемся методом extend. Он принимает в качестве параметра последовательность значений и добавляет их в список по отдельности.

В конце выведем количество полученных записей через метод len, который возвращает количество элементов в последовательности.

import requests

genre1 = input()
genre2 = input()

films = []
for genre in [genre1, genre2]:
  url = f'https://yupest2.pythonanywhere.com/api/v1.0/movies/?genre={genre}'
  r = requests.get(url)
  result = r.json()
  print(result['records'])
  films.extend(result['records'])

print(len(films))

Чтобы сохранить результат в файл, воспользуемся конструкцией:

import json
with open('test.json', 'w', encoding='utf-8') as f:
    json.dump(films, f, ensure_ascii=False)

Здесь вы можете заменить название файла test на свое.

Что такое агрегации?

Для столбчатой, линейной и круговой диаграмм доступен выбор агрегации. Агрегация - это процесс объединения элементов (значений). В процессе агрегации выполняется группировка по некоторому полю (обычно по категории) и применяется арифметическая операция для получения единой количественной величины. Доступные операции: сумма, среднее, максимум, минимум, количество и количество уникальных. Рассмотрим пример агрегаций на таблице:

Дашборд

Дашборд - это отчет, который объединяет диаграммы на одном экране, по-другому его можно назвать - информационной панелью.

Расположенные окна на вкладке Дашборд закреплены за определенным видом диаграммы. Если какая-то диаграмма не будет построена, пустое окно на дашборде удалить не получится.

За верхний край окна перетаскиваются в рабочей области. Чтобы масштабировать окно с диаграммой необходимо потянуть за его нижний правый угол.

Название дашборда можно задать на вкладке Визуализация -> Текст .

Выберите верные утверждения о возможностях платформы(перечислите через пробел):

  1. В платформу можно загрузить данные по ссылке

  2. В платформу можно загрузить данные в форматах xslx, csv, json

  3. Окна с диаграммами на дашборде перетаскиваются за их верхнюю часть

  4. Окна с диаграммами на дашборде перетаскиваются за любую область внутри него

  5. Чтобы масштабировать окно с диаграммой на дашборде, нужно потянуть за нижний правый угол

  6. У окна с диаграммой на дашборде можно потянуть за любую сторону и угол, чтобы изменить его размер

  7. Выбрать операцию агрегирования можно на дашборде

  8. Операцию агрегирования можно выбрать только при построении некоторых диаграмм

Столбчатая диаграмма

Столбчатая диаграмма позволяет наглядно сравнивать значения (категории) по какой-либо метрике (численному значению). Каждый столбец соответствует категории, а размер столбца - числовому значению. 

По умолчанию программа сортирует столбцы по убыванию. Столбчатую диаграмму можно переключить в горизонтальное положение, чтобы ее перевернуть. Функция “Создать топ” позволяет отсечь меньшие столбцы и оставить только n-ое количество (20 шт. и менее).

Также с помощью такого вида диаграммы можно получить распределение некоторых категорий в данных: сколько раз встретились категории. Оценим в каких жанрах фильмы снимаются чаще и реже. Для этого построим столбцы жанров по количеству уникальных фильмов (поле code).

Задача 5. Топ-5 фильмов по количеству оценок

Какие фильмы популярны в обоих базах оценок (Кинопоиск и IMDB)?

В наборе предоставленных данных есть метрики оценок из двух источников - платформ Кинопоиск и IMDB. Критерий “популярности” будем оценивать их количеством. Анализ оценок из двух платформ позволит более объективно судить о популярности фильмов. Найдем фильмы, которые находятся в топ-5 фильмов по количеству оценок в обоих платформах.

Для этого построим топ-5 по количеству оценок IMDB. А затем поменяем показатель на количество оценок Кинопоиска. Применим агрегацию “среднее”, чтобы избежать суммирования оценок по одному фильму в случае, когда он представлен в разных жанрах и в нескольких странах.

Линейный график позволяет сравнить динамику показателей во времени. Чаще всего он строится именно на временных рядах. 

График показывает тенденцию роста затрат на фильмы и спада прибыли, что можно объяснить предположением, что зритель перестал ходить в кино, и предпочитает стриминговые сервисы.

Диаграмма рассеяния 

Диаграмма рассеяния позволяет оценить зависимость между 2 величинами (между осями Х и Y). Если облако точек стремится к форме линии, это говорит о возможной линейной зависимости. Такой тренд достаточно просто предсказывать, он имеет формулу линейной функции y = kx*b, где y и x могут быть например:

  • вес и рост - чем больше рост, тем больше вес
  • время уборки и количество дней без уборки - чем дольше не прибираемся, тем больше уходит время на уборку
  • дистанция и время, за больший период времени пройдем большую дистанцию 

Предположим, что Кинопоиск - платформа для оценки фильмов русского пользователя, а IMDB - зарубежного. Фильмы с происхождением из России имеют малое количество оценок на платформе IMDB.

Облако точек не показало линейной зависимости, что может говорить о разных предпочтениях стран. С другой стороны, количественные показатели оценок на платформах позволяют оценить степень популярности фильмов. Тогда в самом нижнем левом углу будут непопулярные фильмы, а в правом верхнем - популярные одинаково на обеих платформах.

Построим точки по показателям Количества оценок на Кинопоиске и IMDB. В подсказку при наведении курсора добавим название и код фильма. Сначала посмотрите цвет точек по странам, какой вывод можно сделать?

Цвет по категории позволит фильтровать точки. Один клик на значении легенды уберет точки по такой категории с графика, двойной клик - оставит только текущую категорию на графике.

Затем поменяйте цвет точек на среднюю оценку. Верно ли утверждение, что популярные фильмы имеют выше оценку: распределение оценок от низкой к высокой стремится из левого нижнего угла графика к верхнему правому?

Круговая диаграмма

Круговая диаграмма показывает части целого. Круг разделен на сектора, которые соответствуют некоторой категории, а его размер (угол сектора) соответствует некоторой метрике - числовой величине. Фактические числовые величины обычно в круговой диаграмме показаны в процентах, что характеризует долю от целого (всего круга) - от 100%. 

Оптимальное количество секторов в круговой диаграмме - до 9, при большем количестве диаграмма будет “перегружена” информацией и станет тяжело интерпретируемой. Остальные секторы объединены в сектор - “Другое”.

В примере рассмотрены страны производства, у которых максимальные сборы в России и США. То есть производится оценка самых кассовых фильмов по стране происхождения.
