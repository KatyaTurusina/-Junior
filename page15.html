Общая формулировка задания

Компания "Альтернативное будущее" является мировым лидером в области разработки виртуальных ассистентов на основе искусственного интеллекта. Они создают передовые технологии, которые делают жизнь людей более удобной и эффективной. Перед новогодними праздниками компания решает провести смелый эксперимент: пригласить на стажировку школьников, чтобы воспользоваться их уникальными взглядами и идеями. 

Новое задание касается каждого школьника! Оно говорит нет скучным дневниками и направлено на создание личного помощника - виртуального секретаря для учеников школ Эллис. 

Эллис - электронный личный секретарь, встречает учеников приветственным словом утром, поддерживает полезными советами в течении дня, помогает планировать дела, умно распределять время, напоминает об отложенных задачах, приходит на помощь, когда надо сделать выбор. 

Стажерам компании предстоит создать прототип некоторых функций для электронного личного секретаря. Для создания прототипа будущим джуниорам предоставляется доступ к данным некоторого вполне конкретного школьника. Участникам финального задания предстоит решить следующие задачи:

  1. построить дерево возможностей и сценарный граф для диалога с чат-ботом из приведенных в исходном материале фраз, определить кол-во веток диалога, цепочки фраз в диалоге, длиной самой короткой и самой длинной ветки диалога; 
  2. собрать данные, выполнить анализ данных через визуализацию;
  3. обучить цифрового агента Эллис в сервисе DialogFlow;
  4.  программная реализация “умных” функций чат-бота.

Для задач п.1. запланированы два варианта решения: логический путь и программная реализация.

Рассмотрим структурную схему финала подробнее.

Итак, для финала мы раздели все отделы компании AltFuture по ролям с соответствующими компетенциями для решения задач.  Распределение задач по группам на схеме ниже соответствует рекомендуемому количеству человек в команде.

Кроме того, на схеме изображен примерный вариант порядка решения задач. Обратите внимание, на порядок задач. Номер задачи не означает ее решение в первую очередь. Порядок задач на схеме является оптимальным с точки зрения разработчиков. Однако, команде участников можно поменять его по собственному усмотрению в соответствии с выбранной стратегией для достижения максимального возможного результата на финале.

Задачи разных групп тематически связаны между собой. Они показаны на схеме линией. Например программист собирает данные и передает их аналитику для визуализации данных, теоретик создает агента, на основе которого программист реализует для него динамический ответ, а подход к формированию диалога теоретиком может помочь программисту при решении задачи на автоматизированную генерацию диалога.

Схема распределения задач 

В скобках () рядом с задачей указано время, на которое следует ориентироваться участникам финала при желании достичь максимально возможного результата за определенный для финала период времени.

Также у задач указана тематика и уровень сложности в виде звёздочек.

В качестве результата финала станет умный цифровой агент, реализованный в платформе Dialogflow.

По распределению времени, для решения задач каждым участником закладывается 2 часа на получение решения заданий в соответствии с ролью, 30 минут на проверку задач роли,  30 минут на командное тестирование проекта. Итого выделяется 3 часа. Это среднее время выполнения заданий.
